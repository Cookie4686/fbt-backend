# About this project

โปรเจคนี้ทำมาเพื่อเรียนรู้การพัฒนาแอปพลิเคชันด้วยภาษา go โดยมีขอบเขตค่อนข้างกว้างเพื่อจะได้เรียนรู้อย่างครอบคลุม  
โดย**มีไอเดียมาเพื่อใช้ส่วนตัวเป็นหลักไม่เน้นเชิงพาณิชย์** แต่อาจพอมีมูลค่าเชิงพาณิชย์บ้างเล็กน้อยหากนำไปปรับใหม่

โปรเจคยังไม่มีความคืบหน้ามากนัก หากมีความต้องการนำโค้ดไปใช้หรือเอาไอเดียไปปรับสามารถนำไปใช้ได้เลย

# Idea

โปรเจคนี้เป็นแอพทำบัญชี ที่ต้องการนำเอาหลักเกณฑ์บัญชีคู่ และหลักเกณฑ์คงค้าง มาใช้ในชีวิตประจําวัน  
เพื่อเปรียบเทียบรายได้/รายจ่ายในแต่ละเดือน และจัดหมวดหมู่ต่างๆตาม สินทรัพย์ หนี้สิน ทุน ได้

โดยปัจจุบันได้ใช้ spreadsheet เพื่อบันทึกและสรุปรายการบัญชี  
โดยต้องใช้เวลาพอสมควร แต่หากไม่ทำการบันทึกและดูผ่านแอปธนาคาร/การเงินแทน ข้อมูลที่ได้ก็ไม่ละเอียดตามต้องการ  
บางครั้งข้อมูลสรุปที่ได้ก็จะแปลกหน่อย และต้องสรุปรวมทุกธนาคารอีกด้วย

ปัจจุบันก็มีแอพอย่าง make by kbank ซึ่งส่วนตัวไม่เคยใช้เพราะเห็นเป็นแนวจัดงบ (budgeting) มากกว่า  
แต่อาจสร้างงบมาตามบัญชีที่ใช้และจ่ายเงินตามงบไปได้ แต่ก็ต้องมาคิดค่าเสื่อมหรือเงื่อนไขพิเศษแยกต่อ  
และบางครั้งในบิลนึงอาจมีหลายบัญชีก็ได้ เช่น บิล 100 ประกอบด้วย ค่าน้ำ 20, อาหาร 40, ของหวาน 40

# Planned System

## ระบบยืนยันตัวตน

- password
- oauth
- webauthn
- email/phone verification
- mfa เช่น totp

## ระบบบัญชี

- chart of accounts
- transactions
- report

## ระบบบัญชีเฉพาะ

บัญชีที่มีลักษณะเฉพาะโดยค่อยๆเพิ่มไปตามที่ใช้

### ค่าเสื่อม

ใส่วัน เลือกรูปแบบการคำนวณ ไม่มีอะไรมาก  
(ถ้าเป็น unit of production อาจลำบาก แต่คงไม่มีใครใช้ในชีวิตประจำวันเท่าไร)

### เงินลงทุน

อาจมีการบันทึกเป็นรายหุ้นทุน/หุ้นกู้แยกอีก เผื่อจะบันทึกตามมูลค่ายุติธรรม และเผื่อแยกเป็นแบบ PL/OCI ต่อ

## Automate

หากสามารถบันทึกอัตโนมัติได้จะดีมาก  
เช่น สั่งซื้อ 7-11 แล้วนำรายการมาบันทึกเองเลย  
อยากได้สรุปปลายเดือนให้ละเอียดยันรายตัวสินค้าหรือตามแบรนด์ก็ได้  
ส่วนการใช้จ่ายที่ไม่สามารถบันทึกอัตโนมัติได้ก็อาจต้องบันทึกเอง

ในส่วนนี้แน่นอนว่าไม่สามารถทำได้  
แต่อาจทำโมเดลมาดูรูปใบเสร็จแล้วเอาไปบันทึกอัตโนมัติเอา  
หรือไม่ก็ทำพวกคล้ายๆ web scrape ให้ login ไปดึงข้อมูลมา หรือหายิง api ย้อนไปเอา  
ตรงนี้ยังไม่ได้คิดเท่าไรเพราะ scope ส่วนอื่นก็เยอะแล้ว
